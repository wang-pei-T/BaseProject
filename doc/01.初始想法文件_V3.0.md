我想编写一个基础的 BS 项目框架（偏“通用后台管理”方向）。这个框架希望能被其他项目直接使用或继承，并在此基础上持续扩展更多业务模块。

本文档是“想法文档”，只描述功能与模块边界，用于统一方向与讨论口径，不写成正式需求文档，不包含验收标准与排期。

---

## 1. 总体目标与约定

### 1.1 目标

- 提供一套可复用的后台基础能力：租户、组织、用户、认证、角色权限、菜单、字典、审计、日志、文件、通知、配置、运维等
- 多租户作为一等公民：默认以“共享库共享表 + tenantId”落地，并预留隔离形态演进空间
- 模块化扩展：后续业务模块可以复用统一的认证、授权、审计、日志、配置、文件与通知能力

### 1.2 核心概念（统一语言）

- 租户（Tenant）：业务隔离边界，通常对应一个客户/业务实体
- 机构（Organization）：租户内部的组织结构（树形），用于人员归属与数据范围控制
- 用户（User）：登录主体，通常隶属一个主机构，可扩展附属机构/兼职归属
- 角色（Role）：权限集合的载体，可分配给用户
- 权限（Permission）：对“资源 + 操作”的授权单元，例如 user:read、user:disable；可扩展范围/策略
- 菜单（Menu）：用于导航与页面结构展示，可绑定访问所需权限，但菜单不等同于权限本体
- 审计（Audit）：对关键业务操作的可追溯记录（谁、何时、对谁、做了什么、变更了什么）
- 日志（Log）：面向排障与运行监控的系统日志、访问日志、任务日志等
- 消息中心（Message Center）：站内信与其他通知通道的统一收敛与存储入口

### 1.3 删除策略（软删除 vs 真删除）

通俗说明：后台系统里“删掉的数据”往往还要追责、恢复或对账，因此通常不直接物理删除。

- 默认策略：核心业务数据默认软删除（标记删除 + 保留审计轨迹）
  - 适用对象：用户、机构、角色、权限、菜单、字典、配置项、文件元数据、消息等
- 真删除策略：仅对满足条件的数据进行物理删除（可通过运维任务/归档策略执行）
  - 常见前提：已超过保留期、无业务引用、符合法规/安全要求
- 对外表现：软删除数据默认不对业务查询可见；管理员可按权限进行“恢复/彻底删除”（可选）

### 1.4 跨租户与平台管理员（运营/超管）

通俗说明：如果系统要服务多个租户，通常会存在“平台侧的人”负责开租户、处理工单、排障等，他们的权限与入口要和租户内管理员区分开，否则边界会混乱。

- 主体划分
  - 租户主体：租户内用户（含租户管理员），只能在所属租户边界内操作
  - 平台主体：平台管理员/运营人员，可对多个租户进行管理或协助
- 控制台形态
  - 建议独立平台控制台：专门用于租户管理、全局配置、平台运维、工单协助等
  - 租户控制台：租户内的用户、组织、角色权限等日常管理
- 跨租户操作规则（想法层）
  - 平台管理员跨租户操作必须显式选择“目标租户”，并记录目标租户信息
  - 高危操作可要求二次确认/理由填写（可选）
- 审计要求（想法层）
  - 审计记录同时包含：操作者身份、操作者所属主体（平台/租户）、目标租户、目标对象、操作原因（可选）、请求ID
  - 支持按“目标租户”维度回放平台侧操作轨迹（便于租户审计与对账）

## 2. 模块功能清单（想法版）

### 2.1 认证体系（JWT / Session 双支持）

通俗说明：负责“你是谁”的问题，让用户能安全登录，并保持登录状态。

- 登录 / 注销
- 认证方式双支持
  - JWT：支持访问令牌与刷新令牌（可选），支持主动失效（踢下线）、并发登录控制（可选）
  - Session：支持服务端会话存储（内存/Redis 等可选），支持会话失效与续期策略
- 登录安全能力（可配置）
  - 登录失败次数限制与锁定策略
  - 验证码/人机校验能力
  - 与密码策略、账号状态联动（禁用/锁定/到期等）
- 会话与设备管理（可选）
  - 会话列表、下线指定会话、查看最近登录记录
- 扩展点（预留）
  - 第三方登录/单点登录（SSO）、MFA/OTP

### 2.2 用户管理

通俗说明：负责“系统里有哪些人，他们能不能用、怎么分配权限、怎么追溯操作”。

- 用户生命周期
  - 新增、编辑、启用/禁用、锁定/解锁、删除（默认软删除）
  - 重置密码、强制下线、首次登录改密（可选）
- 用户信息
  - 基础资料（姓名、账号、联系方式等）
  - 账号安全信息（最近登录、失败次数、锁定原因等）
  - 归属信息（主机构、附属机构/兼职机构）
- 角色与授权
  - 绑定/解绑角色
  - 查看用户的有效权限集合（便于排查权限问题）
- 历史与审计
  - 用户关键操作进入审计体系，可按用户维度查看审计记录（角色变更、状态变更、重置密码等）

### 2.3 机构管理（组织架构）

通俗说明：负责“公司/部门怎么组织、人归属哪个部门、部门调整如何记录”。

- 机构结构维护
  - 新增、编辑、启用/禁用、删除（默认软删除）
  - 支持树形结构、多级机构、排序、机构编码（可选）
- 人员归属与调整
  - 查看机构下人员
  - 人员在机构之间移动/调整主归属
  - 批量操作（批量移动、批量分配）（可选）
- 组织范围与数据范围基础
  - 基于机构树定义范围：本机构/含下级/自定义机构集合（供数据权限使用）
- 历史与审计
  - 结构与人员调整等关键操作进入审计体系，可按机构维度检索审计记录

### 2.4 租户管理（多租户）

通俗说明：负责“系统服务多个客户时，怎么开通客户、怎么隔离数据、怎么统一运营”。

- 租户生命周期
  - 新增、编辑、启用/停用、到期/续期、删除（默认软删除）
  - 租户状态影响登录与资源访问（停用/到期可限制登录）
- 租户管理员与初始化
  - 创建租户时初始化管理员账号（可选初始化内置角色与菜单）
  - 管理员重置密码、强制下线、权限修复（可选）
- 租户隔离与归属
  - 核心数据与租户绑定（tenantId）
  - 支持租户级配置覆盖（见配置中心）
- 租户级安全与策略（可选）
  - 租户级密码策略、登录策略、会话策略等
- 平台侧协助（可选）
  - 平台管理员为租户排障/协助操作的能力与审计（见 1.4）

### 2.5 角色管理（RBAC）

通俗说明：把权限打包成“角色”，再把角色分给用户，减少逐个授权的复杂度。

- 角色维护
  - 新增、编辑、启用/禁用、删除（默认软删除）
  - 内置角色（系统预置）与自定义角色区分（内置角色可限制变更）
- 角色与权限
  - 为角色配置权限（资源 + 操作）
  - 查看角色下权限明细，支持按模块/资源过滤
- 角色与用户
  - 查看角色下用户列表
  - 批量为用户分配/回收角色（可选）

### 2.6 权限管理（资源 / 操作）

通俗说明：定义“可以对什么对象做什么事”，然后让系统在页面和接口层面统一校验。

- 权限模型
  - 权限包含：唯一标识（code）、名称、描述、所属模块、资源类型、操作类型、状态
  - 支持权限分组/标签（便于管理与授权）
- 范围与策略（可选）
  - 权限可携带范围策略（组织范围、数据范围、条件策略），用于“功能权限 + 数据权限”的组合
- 权限分发与追溯
  - 角色绑定权限、用户通过角色获得权限
  - 支持查看“某用户为何拥有该权限”（用户 -> 角色 -> 权限）
- 权限校验统一化
  - 页面访问、接口访问、操作按钮都基于同一权限体系校验（避免各处散落判断）

### 2.7 菜单管理（多级菜单 + 操作点）

通俗说明：管理后台左侧/顶部的导航结构，以及页面里的按钮操作点如何跟权限挂钩。

- 菜单结构
  - 多级菜单、排序、启用/禁用、隐藏/显示
  - 菜单类型：目录/页面/外链（可选）
- 菜单与权限的关系
  - 菜单项可绑定访问所需权限（无权限则不展示或不可访问）
  - 页面内操作（按钮/动作）作为“操作点”管理，并绑定权限标识
- 个性化（可选）
  - 收藏/常用菜单、最近访问等（也可归入个人设置）

### 2.8 字典管理（业务枚举与下拉）

通俗说明：统一维护业务里的“枚举/下拉选项”，例如状态、类型、级别等，给页面和接口做统一来源。

- 字典类型与字典项
  - 字典类型：唯一编码、名称、描述、状态
  - 字典项：字典类型编码、值、显示名、排序、状态、扩展字段（可选）
- 范围
  - 全局字典与租户字典（租户可覆盖/扩展全局字典）（可选）
- 使用方式
  - 提供统一查询接口，支持缓存与版本刷新（可选）

### 2.9 密码策略

通俗说明：统一规定“密码怎么设才安全、多久换一次、输错多少次锁定”，并让系统强制执行。

- 密码复杂度
  - 长度、字符类型（大小写/数字/特殊字符）、禁止弱密码（可选）
- 轮换与历史
  - 密码有效期、到期提示与强制修改（可选）
  - 历史密码不可重复 N 次（可选）
- 账户安全联动
  - 连续失败锁定、解锁方式、管理员重置流程

### 2.10 个人设置（含通知偏好，不含消息存储）

通俗说明：让用户管理“自己的资料与偏好”。消息本身统一归消息中心存储，这里只放偏好设置，避免两套“个人消息”。

- 个人资料维护（基础信息、头像等）
- 修改密码
- 个人安全设置（可选：MFA、设备管理、登录提醒）
- 通知偏好（可选）
  - 站内信/邮件/短信等通道的订阅开关与免打扰（具体消息见消息中心）
- 个人偏好（可选：主题、语言、默认首页等）

## 3. 审计体系

通俗说明：出了问题能追溯“谁在什么时候做了什么”，并能看到关键字段怎么改的，用于追责、合规与排障。

### 3.1 审计范围

- 覆盖关键管理操作与安全相关操作，例如：
  - 用户：新增/编辑/启停用/重置密码/分配角色/强制下线
  - 机构：新增/编辑/结构调整/人员移动
  - 角色/权限/菜单：新增/编辑/授权变更
  - 租户：新增/停用/到期/初始化管理员
  - 配置变更、字典变更、密码策略变更等
- 覆盖跨租户与平台侧操作
  - 平台管理员对目标租户的任何变更都必须进入审计

### 3.2 审计记录内容（想法层）

- 操作主体：操作人、主体类型（平台/租户）、主体租户（若有）、来源端（可选）、IP、UA
- 目标租户：跨租户操作时记录目标租户（租户内操作可默认等于主体租户）
- 操作对象：对象类型、对象ID、对象名称（可选）
- 操作行为：操作类型（新增/修改/删除/启停用/授权/登录等）
- 变更信息：变更前后差异（建议结构化存储），备注/原因（可选）
- 关联信息：请求ID/追踪ID（用于串联日志与排障）

### 3.3 审计查询与展示

- 按模块/对象类型/对象ID/操作人/时间范围检索
- 支持查看某个用户/机构/角色/租户的审计轨迹
- 审计数据保留与归档策略（可配置/可选）

## 4. 日志体系

通俗说明：用于排障与运营监控的“技术侧记录”，和审计互补：审计偏业务追溯，日志偏运行细节。

### 4.1 日志类型

- 访问日志：请求入口、耗时、状态码、租户、用户、路由、请求ID
- 应用日志：业务关键日志、异常日志
- 安全日志：登录失败、锁定、权限拒绝、敏感操作等（可与审计互补）
- 任务日志：批量任务/异步任务执行过程与结果（可选）

### 4.2 日志能力（想法层）

- 统一日志格式（结构化字段，便于检索）
- TraceId/RequestId 贯穿，支持与审计记录互相跳转定位（可选）
- 日志级别与输出策略可配置（控制台/文件/集中式采集，具体实现视技术栈）

---

## 5. 文件与附件管理

通俗说明：把“上传的文件”做成通用能力，任何业务模块需要附件时都走同一套存储与权限控制。

### 5.1 文件能力

- 上传/下载/预览（视文件类型而定）
- 文件元数据管理
  - 文件名、大小、类型、hash（可选）、上传人、上传时间、归属租户、业务关联信息
- 存储适配
  - 支持本地存储 / 对象存储（OSS/S3 等）等实现可替换

### 5.2 附件与业务关联

- 附件作为通用资源，可关联到任意业务对象（例如用户头像、导入文件、资质附件等）
- 支持附件权限控制（按租户、按对象、按权限点）
- 支持附件清理策略（孤儿文件治理、过期清理等，可选）

## 6. 消息中心（通知统一收敛）

通俗说明：所有“发给人的消息”统一走消息中心存储与查询，避免出现两套站内信/个人消息。

### 6.1 通知类型与通道

- 站内信（默认，统一存储在消息中心）
- 邮件/短信/企业IM 等通道（预留扩展）

### 6.2 消息能力（想法层）

- 消息模板（标题、内容、变量占位）
- 发送策略
  - 立即发送 / 延迟发送 / 批量发送（可选）
  - 失败重试与投递结果记录（可选）
- 用户侧能力
  - 未读/已读、全部已读、按类型筛选、消息详情
- 业务触发
  - 安全提醒（登录异常、密码到期）
  - 管理提醒（账号被禁用、角色变更）
  - 系统公告（可选）
- 个人偏好关系
  - 用户的订阅开关与免打扰归个人设置（2.10），但消息内容与状态归消息中心

## 7. 配置中心（运维热改键值，不承载业务枚举）

通俗说明：配置中心用于“运维/产品想改就改的开关与参数”，比如登录策略开关、阈值、白名单等；字典用于业务下拉枚举，两者不要混用。

### 7.1 配置分类

- 系统级配置：全局生效（例如开关、默认策略）
- 租户级配置：覆盖/继承系统配置（例如登录策略、密码策略、品牌信息）

### 7.2 配置能力（想法层）

- 配置项维护：新增/编辑/启停用、分组、描述
- 配置读取：统一接口、缓存（可选）、版本号/更新时间
- 配置生效方式（可选）
  - 实时生效 / 重启生效 / 灰度生效（预留）
- 配置变更审计
  - 所有配置变更进入审计体系（含平台侧跨租户变更）

### 7.3 与字典的边界约定

- 配置中心：运维热改的键值/参数/阈值/开关，面向运行策略
- 字典管理：业务枚举与下拉选项，面向业务表达与展示

---

## 8. 运维体系

通俗说明：提供“系统在线运行”所需的健康检查、诊断、指标与运维入口，方便部署与排障。

### 8.1 运维入口能力

- 健康检查
  - liveness / readiness（或等价能力），用于部署与编排系统的健康探测
- 指标与监控（可选）
  - 请求量、错误率、耗时分布、慢接口、任务成功率等
- 在线诊断（可选）
  - 查看当前版本信息、配置摘要、依赖连通性（DB/Redis/对象存储）

### 8.2 运行维护能力（可选）

- 缓存管理：按模块清理缓存、刷新字典/菜单/权限快照等
- 任务中心：查看异步任务列表、重试、终止
- 数据修复工具：权限/菜单一致性检查与修复（慎用，按权限控制）
