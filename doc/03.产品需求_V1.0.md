## 1. 文档说明

### 1.1 目的

本 PRD 用于指导产品设计、研发实现与测试验收，按模块写清：功能描述、页面/接口行为、规则、异常、权限、审计点、配置项、验收标准。

### 1.2 范围

- 平台侧（Platform Console）：租户开通与治理、跨租户协助、全局配置、平台审计/日志、基础运维
- 租户侧（Tenant Console）：认证会话、组织用户、角色权限、菜单、字典、文件、消息中心、审计/日志、个人设置、租户级配置

### 1.3 统一约定

- tenantId 是默认隔离边界：租户侧请求隐式绑定 tenantId；平台侧跨租户请求必须显式选择 targetTenantId
- 删除默认软删除；真删除仅作为运维能力且需满足条件
- 权限冲突：拒绝优先（deny > allow），并提供权限追溯解释
- 站内信统一归消息中心存储；个人设置仅存通知偏好，不单独存消息
- 配置中心用于运维热改键值/策略参数；字典用于业务枚举/下拉选项

## 2. 全局规则（跨模块）

### 2.1 多租户边界（平台侧 vs 租户侧）

#### 功能描述

明确平台主体与租户主体的权限与数据边界，避免跨租户越权与误操作。

#### 页面/接口行为

- 平台侧：所有涉及租户数据的页面需提供“目标租户选择器”（列表筛选/顶部选择器），并在界面显式展示当前 targetTenantId
- 租户侧：不提供 targetTenantId 切换能力，所有数据操作仅在本租户范围内

#### 规则

- 租户主体不得对非本租户数据执行读写操作
- 平台主体跨租户操作必须显式选择目标租户
- 平台侧与租户侧建议独立控制台与菜单体系

#### 异常

- 未选择目标租户的跨租户页面/接口：提示“请选择目标租户”
- 目标租户不存在/已停用/已到期：提示并拒绝写操作（读操作按策略配置）

#### 权限

- 平台权限与租户权限分域管理，互不复用同一套角色（允许复用底层权限组件，但不共用角色数据）

#### 审计点

- 平台侧任一跨租户写操作必须记录：operatorType=platform、targetTenantId、操作原因（可选）、requestId

#### 配置项

- platform.console.requireTargetTenantForWrite（默认 true）

#### 验收标准

- 租户侧无法通过任何入口访问/修改其他租户数据
- 平台侧未选择目标租户时无法执行跨租户写操作
- 平台侧跨租户写操作在审计中可按 targetTenantId 查询到完整轨迹

### 2.2 数据隔离（tenantId 强制范围与例外清单）

#### 功能描述

实现默认 tenantId 强隔离，并定义少量“全局数据”的例外类型与读写规则。

#### 页面/接口行为

- 租户侧所有列表/详情接口不暴露 tenantId 参数（由登录上下文决定）
- 平台侧所有面向租户数据的接口必须携带 targetTenantId（路径参数或请求体字段）

#### 规则

- 默认：任何租户数据读写均必须校验 tenantId 范围
- 例外数据类型（仅允许以下类型作为全局域）：
  - 全局字典（可选）：平台维护，租户可覆盖/扩展
  - 全局配置（可选）：平台维护，租户按策略覆盖
  - 权限点定义（可选）：全局定义，租户侧仅进行绑定使用
  - 平台审计/平台日志：平台域数据，但记录 targetTenantId

#### 异常

- 任一请求缺失 tenantId 上下文：拒绝并提示“租户上下文缺失”
- 平台侧请求 targetTenantId 非法：拒绝并提示“目标租户无效”

#### 权限

- 例外数据的读写权限必须显式配置（platform/global/tenant 三域）

#### 审计点

- 涉及例外数据的变更必须记录 dataDomain 与影响范围（global / tenant override）

#### 配置项

- tenant.enforcement.enabled（默认 true）
- tenant.exception.dataDomains（默认仅允许上述清单类型）

#### 验收标准

- 在未显式列入例外清单的数据类型上，无法出现跨租户读写
- 平台审计/日志中可按 targetTenantId 检索到跨租户操作与访问记录

### 2.3 权限冲突规则（允许/拒绝优先级）

#### 功能描述

定义权限判定的一致规则，避免不同模块出现“同一权限点判定不一致”。

#### 页面/接口行为

- 页面加载时按权限决定菜单可见与页面可访问
- 页面内操作点（按钮/动作）按权限点控制可见/可用
- 无权限的接口请求返回统一错误（403），并给出可追溯的拒绝原因（对外展示可简化）

#### 规则

优先级从高到低：

1) 边界拒绝：租户主体跨租户直接拒绝
2) 安全状态拒绝：禁用/锁定/到期/强制改密等优先拦截
3) 显式拒绝（deny）优先于显式允许（allow）
4) allow 之间按细粒度优先（更具体资源、更严格条件优先决定最终范围）

deny 的落点（V1 约定）：

- V1.0 支持“策略黑名单型拒绝”：例如禁止某些高危操作、禁止特定来源端等
- 不做“用户级别 deny 权限点配置 UI”（后续版本可扩展）

#### 异常

- 权限点不存在：拒绝并记录安全日志，提示“权限配置异常”
- 权限追溯链路不完整：返回拒绝并记录异常日志

#### 权限

- 权限点管理：platform 侧可维护（可选），租户侧仅绑定使用
- 权限追溯查询：租户管理员可查看本租户用户权限追溯；平台超管可跨租户查看

#### 审计点

- 权限相关变更（角色授权、菜单绑定权限、权限点元数据变更）必须审计
- 权限拒绝事件进入安全日志（可选进入审计）

#### 配置项

- authz.denyPolicy.enabled（默认 true）
- authz.explain.enabled（默认 true）

#### 验收标准

- 同一权限点在页面与接口校验结果一致
- 存在 deny 策略时，即使有 allow，也能稳定拒绝并输出可追溯原因

### 2.4 删除与恢复（软删恢复冲突处理）

#### 功能描述

提供软删除与恢复能力，并对恢复时的唯一键冲突给出明确处理规则。

#### 页面/接口行为

- 默认列表不展示已删除数据
- 管理页面提供“回收站/包含已删除”筛选（仅管理员可见）
- 恢复操作在 UI 上需要二次确认，并展示可能的冲突检测结果

#### 规则

- 默认软删除：删除标记 + 删除时间 + 删除人
- 恢复前必须进行冲突校验（账号、编码等唯一性字段）
- 冲突默认处理：拒绝恢复并提示冲突字段
- 真删除：仅在满足保留期、无引用、具备权限的情况下可执行（运维能力）

#### 异常

- 恢复冲突：返回冲突字段列表
- 存在强引用：拒绝删除并提示引用来源（若可获取）

#### 权限

- 删除：对应资源的 delete 权限
- 恢复：对应资源的 restore 权限（默认仅管理员）
- 真删除：purge 权限（默认仅平台超管/运维）

#### 审计点

- 删除、恢复、真删除均必须审计；恢复需记录冲突检测结果与处理方式

#### 配置项

- deletion.mode.default=soft
- deletion.restore.conflictPolicy=reject（默认）

#### 验收标准

- 被软删的数据不在默认查询范围内
- 恢复时遇到唯一键冲突能稳定拒绝并提示冲突项
- 删除/恢复/真删除均可在审计中追溯到操作者与对象

### 2.5 审计留存与归档

#### 功能描述

对关键操作提供可追溯记录，并支持按保留策略归档，兼顾合规与成本。

#### 页面/接口行为

- 审计列表支持：模块/对象类型/对象ID/操作者/时间范围/目标租户筛选（平台侧）
- 审计详情展示：基础信息 + 变更差异（结构化）+ 关联 requestId

#### 规则

- 默认保留：
  - 业务审计：≥ 180 天（默认）
  - 安全审计：≥ 1 年（默认）
- 归档触发（可选）：
  - 超过保留阈值按时间归档
  - 达到容量阈值滚动归档

#### 异常

- 归档后明细不可用：需提示“已归档”，并给出查询指引（如走归档入口）

#### 权限

- 租户侧：仅能查看本租户审计
- 平台侧：可跨租户查看（需平台权限）

#### 审计点

- 审计策略变更（保留期、归档阈值）必须审计

#### 配置项

- audit.retention.days.default=180
- audit.retention.days.security=365
- audit.archive.enabled（默认 false）
- audit.archive.trigger.days / audit.archive.trigger.size（可选）

#### 验收标准

- 审计记录包含 operatorType、tenantId/targetTenantId、requestId、对象信息、操作类型、差异信息
- 达到保留/归档条件时按策略执行，且查询行为符合预期提示

### 2.6 配置中心（实时生效一致性策略）

#### 功能描述

提供可运维热改的键值配置，支持平台全局配置与租户覆盖配置，并确保变更的生效一致性可控。

#### 页面/接口行为

- 配置项维护：列表/新增/编辑/启停用/查看历史版本（可选）
- 配置读取：业务模块通过统一接口读取（对用户界面不可见）

#### 规则

- 配置项必须包含：key、value、type、scope（platform/tenant）、启用状态、revision
- 生效一致性分级：
  - 强一致：变更后对新请求立即生效（高风险开关）
  - 最终一致：允许短窗口延迟生效（普通参数）
- 租户覆盖优先于平台默认（若允许覆盖）

#### 异常

- key 冲突：拒绝新增，提示“配置 key 已存在”
- value 类型不匹配：拒绝保存并提示期望类型

#### 权限

- 平台全局配置：平台管理员可维护
- 租户覆盖配置：租户管理员可维护（在允许覆盖范围内）

#### 审计点

- 配置新增/修改/启停用必须审计，记录旧值/新值、revision、影响范围

#### 配置项

- config.refresh.strategy=polling（默认）/event（可选）
- config.refresh.pollIntervalMs（默认值可选）
- config.refresh.strongConsistency.enabled（默认 true）

#### 验收标准

- 修改配置后，强一致配置在可接受时间内对新请求生效
- 配置变更的 revision 可被查询，且审计中可追溯旧值/新值与影响范围

## 3. 平台侧（Platform Console）模块需求

### 3.1 租户管理

#### 功能描述

平台侧对租户进行开通、治理与生命周期管理，并初始化租户管理员账号。

#### 页面/接口行为

- 页面
  - 租户列表：搜索（名称/编码/状态）、筛选（启用/停用/到期）、分页
  - 租户详情：基础信息、到期信息、管理员信息、最近变更
  - 新增/编辑租户：输入基础信息、有效期、初始化管理员（可选）
  - 操作：启用/停用、续期、重置租户管理员密码、强制下线管理员、软删除、恢复（可选）
- 接口（示例）
  - GET /platform/tenants
  - POST /platform/tenants
  - GET /platform/tenants/{tenantId}
  - PATCH /platform/tenants/{tenantId}
  - POST /platform/tenants/{tenantId}:enable|disable|renew
  - POST /platform/tenants/{tenantId}/admin:resetPassword|forceLogout
  - DELETE /platform/tenants/{tenantId}（软删）
  - POST /platform/tenants/{tenantId}:restore（可选）

#### 规则

- 租户停用/到期后：
  - 默认禁止租户主体登录
  - 平台主体仍可进入租户上下文进行必要治理（需权限与审计）
- 租户编码全局唯一（若启用编码）
- 删除默认软删；恢复需冲突校验（编码/域名等）

#### 异常

- 租户已到期：提示“租户已到期”，阻止租户侧登录
- 初始化管理员账号冲突：提示冲突并要求调整

#### 权限

- platform.tenant.manage
- platform.tenant.admin.resetPassword / forceLogout
- platform.tenant.delete / restore（可选）

#### 审计点

- 创建/编辑/启停用/续期/删除/恢复
- 初始化管理员、重置密码、强制下线

#### 配置项

- tenant.expire.behavior=denyLogin（默认）
- tenant.default.admin.role（默认：租户管理员）

#### 验收标准

- 平台可完成租户全生命周期操作且均有审计记录
- 停用/到期租户无法在租户侧登录（按配置策略）

### 3.2 平台账号与权限（可选）

#### 功能描述

管理平台管理员账号、平台域角色与权限，保障平台操作可控。

#### 页面/接口行为

- 页面：平台账号列表/详情/新增/编辑/禁用、平台角色授权（可选）
- 接口（示例）：/platform/users、/platform/roles、/platform/permissions（按平台域）

#### 规则

- 平台超管账号不可被普通运营禁用
- 平台侧权限与租户侧权限分域隔离

#### 异常

- 禁用最后一个平台超管：拒绝并提示

#### 权限

- platform.user.manage
- platform.role.manage

#### 审计点

- 平台账号状态变更、角色授权变更

#### 配置项

- platform.superAdmin.minCount=1

#### 验收标准

- 平台权限变更可追溯，且无法删除/禁用导致平台失管

### 3.3 跨租户协助（可选）

#### 功能描述

平台运营在目标租户上下文中执行受控协助动作，用于排障与客户支持。

#### 页面/接口行为

- 页面：目标租户选择器 + 协助动作面板（只展示被允许的动作）
- 协助动作示例：强制下线某用户、重置某用户密码（可选）、查看权限追溯、查看审计
- 接口（示例）：POST /platform/assist/{tenantId}/users/{userId}:forceLogout

#### 规则

- 必须显式选择目标租户
- 高危操作要求填写原因（可配置）
- 协助动作集合由平台配置白名单控制

#### 异常

- 目标用户不存在：提示并拒绝
- 协助动作不在白名单：拒绝并记录安全日志

#### 权限

- platform.assist.use
- platform.assist.action.*（按动作拆分）

#### 审计点

- 所有协助动作必须审计并记录 targetTenantId 与原因（若启用）

#### 配置项

- platform.assist.requireReason（默认 true）
- platform.assist.allowedActions（默认最小集合）

#### 验收标准

- 平台协助动作只能在目标租户上下文中执行，且全量可审计

### 3.4 平台审计与平台日志

#### 功能描述

平台侧可查询平台域审计与日志，并按目标租户回溯跨租户操作轨迹。

#### 页面/接口行为

- 页面：平台审计列表/详情、平台日志列表（访问/安全/异常，视实现）
- 接口（示例）：GET /platform/audits、GET /platform/logs

#### 规则

- 平台日志与平台审计属于平台域数据，但必须包含 targetTenantId（若与租户相关）

#### 异常

- 查询范围过大：提示缩小时间范围

#### 权限

- platform.audit.read
- platform.log.read

#### 审计点

- 对审计/日志策略的变更（保留期、归档等）

#### 配置项

- audit.retention.*（见全局规则）
- log.retention.*（可选）

#### 验收标准

- 平台可按 targetTenantId 检索到跨租户写操作的审计记录

## 4. 租户侧（Tenant Console）模块需求

### 4.1 认证与会话（JWT / Session 双支持）

#### 功能描述

提供统一登录、注销与会话管理能力，支持 JWT 与 Session 两种模式切换或并存。

#### 页面/接口行为

- 页面
  - 登录页：账号 + 密码（可选验证码）
  - 登出：清理当前会话
  - 会话管理（可选）：查看当前账号所有会话并可下线
- 接口（示例）
  - POST /auth/login（支持 mode=jwt|session）
  - POST /auth/logout
  - GET /auth/me
  - GET /auth/sessions（可选）
  - POST /auth/sessions/{sessionId}:revoke（可选）

#### 规则

- 登录失败次数达到阈值后锁定（可配置）
- 禁用/锁定/到期账号禁止登录
- JWT 模式支持刷新令牌（可选）

#### 异常

- 密码错误：提示“账号或密码错误”，不泄露账号是否存在
- 锁定：提示“账号已锁定”，并提示解锁方式（若配置）

#### 权限

- 登录不需要业务权限；会话管理需要 user.session.manage（可选）或仅对本人开放

#### 审计点

- 登录成功/失败、登出、强制下线（安全审计/安全日志）

#### 配置项

- auth.mode=jwt|session|both
- auth.login.failLock.threshold / duration
- auth.captcha.enabled

#### 验收标准

- JWT/Session 模式按配置可用且互不干扰
- 登录失败锁定策略生效，且记录安全审计/日志

### 4.2 用户管理

#### 功能描述

维护租户内用户账号、状态、安全信息、角色分配，并支持软删除与恢复（如启用回收站）。

#### 页面/接口行为

- 页面
  - 用户列表：搜索（姓名/账号/手机号/邮箱）、筛选（状态/机构/角色）、分页
  - 用户详情：基础信息、所属机构、角色、最近登录、安全状态
  - 新增/编辑用户
  - 操作：启用/禁用、锁定/解锁、重置密码、强制下线、软删除、恢复（可选）
- 接口（示例）
  - GET /tenant/users
  - POST /tenant/users
  - GET /tenant/users/{userId}
  - PATCH /tenant/users/{userId}
  - POST /tenant/users/{userId}:enable|disable|lock|unlock
  - POST /tenant/users/{userId}:resetPassword|forceLogout
  - DELETE /tenant/users/{userId}（软删）
  - POST /tenant/users/{userId}:restore（可选）

#### 规则

- 账号（登录名/手机号/邮箱）在租户内唯一（按选用字段）
- 禁用用户无法登录；锁定用户按策略解锁
- 重置密码可要求用户下次登录强制改密（可配置）

#### 异常

- 删除自己：默认拒绝（可配置）
- 禁用/删除最后一个租户管理员：默认拒绝（可配置）

#### 权限

- tenant.user.read / create / update / delete / restore（可选）
- tenant.user.resetPassword / forceLogout
- tenant.user.role.assign

#### 审计点

- 新增/编辑/启停用/锁定解锁/重置密码/强制下线/删除/恢复/角色变更

#### 配置项

- tenant.admin.minCount=1
- user.selfDelete.allowed（默认 false）
- user.resetPassword.forceChangeOnNextLogin（默认 true）

#### 验收标准

- 用户增删改查与状态变更正常
- 高风险规则（最后一个管理员、删自己）按策略拦截
- 用户相关关键操作在审计中可查询

### 4.3 机构管理（组织架构）

#### 功能描述

维护租户内组织机构树，支持人员归属调整，并为数据权限提供组织范围基础。

#### 页面/接口行为

- 页面
  - 机构树：新增/编辑/移动/排序（可选）
  - 机构详情：机构信息、下属机构、人员列表
  - 人员调整：移动人员到其他机构；「主归属」在 V1 与单一 `orgId` 等价（无多机构主从字段，多机构延后）
- 接口（示例）
  - GET /tenant/orgs/tree
  - POST /tenant/orgs
  - PATCH /tenant/orgs/{orgId}
  - POST /tenant/orgs/{orgId}:enable|disable
  - DELETE /tenant/orgs/{orgId}（软删）
  - POST /tenant/orgs/{orgId}:restore（可选）
  - POST /tenant/orgs/{orgId}/users:move

#### 规则

- 机构为树结构，禁止形成环
- 删除机构前需处理人员归属（默认拒绝删除或要求先迁移）
- 人员移动需记录前后机构信息（用于审计）

#### 异常

- 将机构移动到其子机构：拒绝
- 删除存在下级机构的机构：拒绝或要求先处理（按策略）

#### 权限

- tenant.org.read / create / update / delete / restore（可选）
- tenant.org.user.move

#### 审计点

- 机构新增/编辑/启停用/结构调整/删除/恢复、人员移动

#### 配置项

- org.delete.requireEmptyChildren（默认 true）
- org.delete.requireNoUsers（默认 true）

#### 验收标准

- 机构树维护正确，人员移动后列表与归属一致
- 结构调整与人员移动均可审计回放

### 4.4 角色管理（RBAC）

#### 功能描述

维护租户内角色，并为角色配置权限集合，支持用户分配角色。

#### 页面/接口行为

- 页面
  - 角色列表/新增/编辑/启停用/软删除
  - 角色授权：按模块/资源选择权限点
  - 角色成员：查看绑定的用户并支持批量调整（可选）
- 接口（示例）
  - GET /tenant/roles
  - POST /tenant/roles
  - PATCH /tenant/roles/{roleId}
  - POST /tenant/roles/{roleId}/permissions:replace
  - GET /tenant/roles/{roleId}/users

#### 规则

- 内置角色不可删除（可选），或删除需特殊权限
- 角色授权变更应触发权限缓存刷新（实现细节，但产品需保证生效及时）

#### 异常

- 删除被引用角色：默认拒绝或要求先解绑（按策略）

#### 权限

- tenant.role.read / create / update / delete
- tenant.role.permission.assign

#### 审计点

- 角色新增/编辑/启停用/删除、角色授权变更、角色成员变更（可选）

#### 配置项

- role.builtin.protect.enabled（默认 true）

#### 验收标准

- 角色授权后用户权限生效符合冲突规则
- 角色授权变更在审计中可追溯差异

### 4.5 权限管理（权限点元数据与追溯）

#### 功能描述

管理权限点元数据（若租户侧允许查看/辅助管理），并提供用户权限追溯能力。

#### 页面/接口行为

- 页面
  - 权限点列表：按模块/资源筛选
  - 权限追溯：输入用户，查看其权限来源与拒绝原因
- 接口（示例）
  - GET /tenant/permissions（只读为主）
  - GET /tenant/users/{userId}/permissions
  - GET /tenant/users/{userId}/permissions:explain

#### 规则

- 权限点定义可由平台侧维护为全局（推荐），租户侧仅使用
- 权限追溯必须能够定位到：来自哪个角色、是否被策略拒绝

#### 异常

- 权限点缺失：提示“权限配置异常”，并记录日志

#### 权限

- tenant.permission.read
- tenant.permission.explain

#### 审计点

- 权限点元数据变更（如存在租户侧编辑）必须审计（默认由平台侧审计）

#### 配置项

- authz.explain.enabled（见全局规则）

#### 验收标准

- 能输出用户权限清单与来源追溯，并包含拒绝原因（若被拒绝）

### 4.6 菜单管理（含操作点）

#### 功能描述

维护租户侧导航菜单结构，并将菜单与权限点绑定，控制页面可见与可访问；维护页面内操作点与权限绑定。

#### 页面/接口行为

- 页面
  - 菜单树：新增/编辑/排序/启停用/隐藏
  - 菜单权限绑定：设置访问所需权限点
  - 操作点管理：为页面配置按钮/动作，并绑定权限点
- 接口（示例）
  - GET /tenant/menus/tree
  - POST /tenant/menus
  - PATCH /tenant/menus/{menuId}
  - POST /tenant/menus/{menuId}/permission:bind
  - GET /tenant/menus:my（返回当前用户可见菜单）

#### 规则

- 菜单不等同于权限点；权限点用于接口与按钮级校验
- 用户无访问权限的菜单项默认不展示；即便展示也必须在路由/接口层校验

#### 异常

- 菜单绑定不存在的权限点：拒绝并提示

#### 权限

- tenant.menu.read / create / update / delete
- tenant.menu.permission.bind

#### 审计点

- 菜单新增/编辑/启停用/排序调整/删除、菜单权限绑定、操作点绑定权限

#### 配置项

- menu.visibility.noPermission=hide（默认）

#### 验收标准

- 不同权限用户看到的菜单与操作点符合权限配置
- 菜单与操作点变更可审计

### 4.7 字典管理（业务枚举）

#### 功能描述

统一维护业务枚举与下拉选项，供页面展示与接口校验使用。

#### 页面/接口行为

- 页面
  - 字典类型列表/新增/编辑/启停用
  - 字典项列表/新增/编辑/排序/启停用
  - （可选）租户覆盖：查看全局字典并覆盖/扩展
- 接口（示例）
  - GET /tenant/dicts/types
  - POST /tenant/dicts/types
  - GET /tenant/dicts/{typeCode}/items
  - POST /tenant/dicts/{typeCode}/items

#### 规则

- 字典类型 code 在范围内唯一（租户内或全局，按设计）
- 字典项 value 在同一 type 下唯一
- 字典用于业务枚举，不承载运行策略键值（配置中心负责）

#### 异常

- code/value 冲突：拒绝并提示

#### 权限

#### 验收标准

- 字典查询接口稳定输出，页面下拉与状态展示一致

### 4.8 文件与附件管理

#### 功能描述

提供统一文件上传/下载/预览与附件关联能力，任何业务对象需要附件时都走同一套能力。

#### 页面/接口行为

- 页面
  - 文件上传组件：上传进度、失败重试（可选）
  - 附件列表：查看、下载、预览、删除（软删）
- 接口（示例）
  - POST /tenant/files（上传）
  - GET /tenant/files/{fileId}
  - GET /tenant/files/{fileId}:download
  - POST /tenant/attachments:bind（绑定到业务对象）
  - GET /tenant/attachments?bizType=&bizId=

#### 规则

- 文件元数据包含：上传人、上传时间、tenantId、内容类型、大小
- 附件需记录业务关联信息（bizType、bizId）
- 下载权限校验：必须满足租户与业务对象访问权限（至少 tenantId）

#### 异常

- 文件过大/类型不允许：拒绝并提示
- 附件绑定对象不存在：拒绝并提示

#### 权限

- tenant.file.upload / download / preview
- tenant.attachment.bind / delete

#### 审计点

- 文件上传、附件绑定/解绑、附件删除、敏感文件下载（可选）

#### 配置项

- file.maxSizeMb
- file.allowedTypes
- file.storage.provider=local|objectStorage

#### 验收标准

- 上传/下载/预览可用，且权限控制正确
- 附件能与业务对象关联并可追溯上传与绑定操作

### 4.9 消息中心（站内信统一存储）

#### 功能描述

统一存储与投递站内消息，并预留扩展到邮件/短信等通道的能力；个人设置仅管理通知偏好。

#### 页面/接口行为

- 页面
  - 消息列表：未读/已读、类型筛选、分页
  - 消息详情：查看内容并自动标记已读（可配置）
  - 操作：标记已读、全部已读、删除（软删，可选）
- 接口（示例）
  - GET /tenant/messages
  - GET /tenant/messages/{messageId}
  - POST /tenant/messages/{messageId}:read
  - POST /tenant/messages:readAll

#### 规则

- 站内信必须统一落在消息中心存储
- 个人设置只存“通知偏好”，不存消息内容与未读状态

#### 异常

- 消息不存在/无权访问：提示并拒绝

#### 权限

- tenant.message.read（对本人默认放行）
- tenant.message.manage（管理端可选）

#### 审计点

- 管理端发送公告/系统通知（若有）必须审计
- 用户阅读行为默认不审计（可选进日志）

#### 配置项

- message.read.markOnOpen=true|false
- message.retention.days（可选）

#### 验收标准

- 消息未读/已读状态正确，且不会出现两套站内信存储

### 4.10 个人设置（含通知偏好）

- 个人资料可修改，修改密码规则正确且有安全审计

### 4.11 审计查询（租户内）
