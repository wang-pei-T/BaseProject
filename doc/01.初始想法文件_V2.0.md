我想编写一个基础的 BS 项目框架（偏“通用后台管理”方向）。这个框架希望能被其他项目直接使用或继承，并在此基础上持续扩展更多业务模块。

本文档是“想法文档”，仅描述要具备的功能与模块边界，不写成正式需求文档，不包含验收口径与排期。

---

## 1. 总体目标与边界

### 1.1 目标

- 提供一套可复用的后台基础能力：用户、组织、租户、角色权限、菜单、字典、审计、日志、认证、运维等
- 支持多租户场景，可在不同隔离策略下落地（以“共享库共享表 + tenantId”作为默认形态）
- 支持后续项目按模块化方式扩展：新增业务模块时能复用统一的认证、授权、审计、日志、配置、文件、通知等基础能力

### 1.2 核心概念（统一语言）

- 租户（Tenant）：业务隔离的边界单位，通常对应一个客户/业务实体
- 机构（Organization）：租户内部的组织结构（树形），用于归属与数据范围控制
- 用户（User）：登录主体，可隶属于一个主机构并支持附属机构/兼职归属
- 角色（Role）：权限集合的载体，可分配给用户
- 权限（Permission）：对“资源 + 操作”的授权单元，例如 user:read、user:disable；可带范围/策略
- 菜单（Menu）：仅用于导航与页面结构展示，可绑定访问所需权限，但菜单不等同于权限
- 审计（Audit）：对关键业务操作的可追溯记录（谁、何时、对谁、做了什么、变更了什么）
- 日志（Log）：面向排障与运行监控的系统日志、访问日志、任务日志等

## 2. 模块功能清单（想法版）

### 2.1 认证体系（JWT / Session 双支持）

- 登录 / 注销
- 认证方式双支持
  - JWT：支持访问令牌与刷新令牌（可选），支持主动失效（踢下线）、并发登录控制（可选）
  - Session：支持服务端会话存储（内存/Redis 等可选），支持会话失效与续期策略
- 登录安全能力
  - 登录失败次数限制与锁定策略（可配置）
  - 验证码/人机校验能力（可配置）
  - 密码策略与账号状态联动（禁用/锁定/到期等）
- 多端与设备管理（可选）
  - 设备/会话列表、下线指定会话、查看最近登录记录
- 对外扩展接口（预留）
  - 第三方登录/单点登录（SSO）、MFA/OTP 等扩展点

### 2.2 用户管理

- 用户生命周期
  - 新增、编辑、启用/禁用、锁定/解锁、删除（建议软删除）
  - 重置密码、强制下线、首次登录改密（可选）
- 用户信息
  - 基础资料（姓名、账号、联系方式等）
  - 账号安全信息（最近登录时间、失败次数、锁定原因等）
  - 归属信息（主机构、附属机构/兼职机构）
- 角色与授权
  - 绑定/解绑角色
  - 支持查看用户的有效权限集合（便于排查权限问题）
- 历史与审计
  - 用户相关关键操作统一进入审计体系，可查看用户维度的审计记录（例如角色变更、状态变更、重置密码等）

### 2.3 机构管理（组织架构）

- 机构结构维护
  - 新增、编辑、启用/禁用、删除（建议软删除）
  - 支持树形结构、多级机构、排序、机构编码（可选）
- 人员归属与调整
  - 查看机构下人员
  - 人员在机构之间移动/调整主归属
  - 支持批量操作（批量移动、批量分配）
- 组织范围与数据范围（为权限体系提供基础）
  - 支持基于机构树定义范围：本机构/含下级/自定义机构集合（供数据权限使用）
- 历史与审计
  - 机构结构与人员调整等关键操作进入审计体系，可按机构维度检索审计记录

### 2.4 租户管理（多租户）

- 租户生命周期
  - 新增、编辑、启用/停用、到期/续期、删除（建议软删除）
  - 租户状态影响登录与资源访问（停用/到期可限制登录）
- 租户管理员与初始化
  - 创建租户时初始化管理员账号（可选初始化内置角色与菜单）
  - 管理员重置密码、强制下线、权限修复（可选）
- 租户隔离与归属
  - 所有核心数据与租户绑定（tenantId）
  - 支持按租户维度的配置覆盖（见配置中心）
- 租户级安全与策略（可选）
  - 租户级密码策略、登录策略、会话策略等
- 租户审计与统计（可选）
  - 按租户查看关键操作审计、登录概览、资源使用概览等

### 2.5 角色管理（RBAC）

- 角色维护
  - 新增、编辑、启用/禁用、删除
  - 内置角色（系统预置）与自定义角色区分（内置角色可限制变更）
- 角色与权限
  - 为角色配置权限（资源 + 操作）
  - 支持查看角色下的权限明细，支持按模块/资源过滤
- 角色与用户
  - 查看角色下的用户列表
  - 批量为用户分配/回收角色（可选）

### 2.6 权限管理（资源 / 操作）

- 权限模型
  - 每个权限包含：唯一标识（code）、名称、描述、所属模块、资源类型、操作类型、状态
  - 支持权限分组/标签（便于管理与授权）
- 范围与策略（可选）
  - 权限可携带范围策略（如组织范围、数据范围、条件策略），用于“功能权限 + 数据权限”的组合
- 权限分发
  - 角色绑定权限、用户通过角色获得权限
  - 支持查看“某用户为何拥有该权限”（追溯链路：用户 -> 角色 -> 权限）
- 权限校验统一化
  - 页面访问、接口访问、操作按钮都基于同一权限体系进行校验（避免各处散落的判断逻辑）

### 2.7 菜单管理（多级菜单 + 操作点）

- 菜单结构
  - 多级菜单、排序、启用/禁用、隐藏/显示
  - 菜单类型：目录/页面/外链（可选）
- 菜单与权限的关系
  - 菜单项可绑定访问所需权限（无权限则不展示或不可访问）
  - 页面内操作（按钮/动作）建议作为“操作点”管理，并绑定权限标识
- 个性化（可选）
  - 收藏/常用菜单、最近访问等（也可归入个人设置）

### 2.8 字典管理

- 字典类型与字典项
  - 字典类型：唯一编码、名称、描述、状态
  - 字典项：字典类型编码、值、显示名、排序、状态、扩展字段（可选）
- 范围
  - 支持全局字典与租户字典（租户可覆盖/扩展全局字典，可选）
- 使用方式
  - 提供统一查询接口，支持缓存与版本刷新（可选）

### 2.9 密码策略

- 密码复杂度
  - 长度、字符类型（大小写/数字/特殊字符）、禁止弱密码（可选）
- 轮换与历史
  - 密码有效期、到期提示与强制修改（可选）
  - 历史密码不可重复 N 次（可选）
- 账户安全联动
  - 连续失败锁定、解锁方式、管理员重置流程

### 2.10 个人设置

- 个人资料维护（基础信息、头像等）
- 修改密码
- 个人安全设置（可选：MFA、设备管理、登录提醒）
- 个人偏好（可选：主题、语言、默认首页等）

## 3. 审计体系（新增）

### 3.1 审计范围

- 覆盖关键管理操作与安全相关操作，例如：
  - 用户：新增/编辑/启停用/重置密码/分配角色/强制下线
  - 机构：新增/编辑/结构调整/人员移动
  - 角色/权限/菜单：新增/编辑/授权变更
  - 租户：新增/停用/到期/初始化管理员
  - 配置变更、字典变更、密码策略变更等

### 3.2 审计记录内容（想法层）

- 操作主体：操作人、所属租户、来源端（可选）、IP、UA
- 操作对象：对象类型、对象ID、对象名称（可选）
- 操作行为：操作类型（新增/修改/删除/启停用/授权/登录等）
- 变更信息：变更前后差异（建议结构化存储），备注/原因（可选）
- 关联信息：请求ID/追踪ID（用于串联日志与排障）

### 3.3 审计查询与展示

- 按模块/对象类型/对象ID/操作人/时间范围检索
- 支持查看某个用户/机构/角色的审计轨迹
- 审计数据保留与归档策略（可配置/可选）

---

## 4. 日志体系（新增）

### 4.1 日志类型

- 访问日志：请求入口、耗时、状态码、租户、用户、路由、请求ID
- 应用日志：业务关键日志、异常日志
- 安全日志：登录失败、锁定、权限拒绝、敏感操作等（可与审计互补）
- 任务日志：批量任务/异步任务执行过程与结果（可选）

### 4.2 日志能力（想法层）

- 统一日志格式（结构化字段，便于检索）
- TraceId/RequestId 贯穿，支持与审计记录互相跳转定位（可选）
- 日志级别与输出策略可配置（控制台/文件/集中式采集，具体实现视技术栈）

---

## 5. 文件与附件管理（新增）

### 5.1 文件能力

- 上传/下载/预览（视文件类型而定）
- 文件元数据管理
  - 文件名、大小、类型、hash（可选）、上传人、上传时间、归属租户、业务关联信息
- 存储适配
  - 支持本地存储 / 对象存储（OSS/S3 等）等实现可替换

### 5.2 附件与业务关联

- 附件作为通用资源，可关联到任意业务对象（例如用户头像、导入文件、资质附件等）
- 支持附件权限控制（按租户、按对象、按权限点）
- 支持附件清理策略（孤儿文件治理、过期清理等，可选）

## 6. 消息通知（新增）

### 6.1 通知类型与通道

- 站内消息（默认）
- 邮件/短信/企业IM 等通道（预留扩展）

### 6.2 通知能力（想法层）

- 消息模板（标题、内容、变量占位）
- 发送策略
  - 立即发送 / 延迟发送 / 批量发送（可选）
  - 失败重试与投递结果记录（可选）
- 用户侧能力
  - 未读/已读、全部已读、按类型筛选、消息详情
- 业务触发
  - 安全提醒（登录异常、密码到期）
  - 管理提醒（账号被禁用、角色变更）
  - 系统公告（可选）

## 7. 配置中心（新增）

### 7.1 配置分类

- 系统级配置：全局生效（例如开关、默认策略）
- 租户级配置：覆盖/继承系统配置（例如登录策略、密码策略、品牌信息）

### 7.2 配置能力（想法层）

- 配置项维护：新增/编辑/启停用
- 配置读取：统一接口、缓存（可选）、版本号/更新时间
- 配置生效方式（可选）
  - 实时生效 / 重启生效 / 灰度生效（预留）
- 配置变更审计：所有配置变更进入审计体系

---

## 8. 运维体系（新增）

### 8.1 运维入口能力

- 健康检查
  - liveness / readiness（或等价能力），用于部署与编排系统的健康探测
- 指标与监控（可选）
  - 请求量、错误率、耗时分布、慢接口、任务成功率等
- 在线诊断（可选）
  - 查看当前版本信息、配置摘要、依赖连通性（DB/Redis/对象存储）

### 8.2 运行维护能力（可选）

- 缓存管理：按模块清理缓存、刷新字典/菜单/权限快照等
- 任务中心：查看异步任务列表、重试、终止
- 数据修复工具：权限/菜单一致性检查与修复（慎用，按权限控制）
