# BaseProject 全量开发阶段执行方案 V1.0

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 基于 `doc` 与 `prototypes` 完成前后端全量开发，前端页面与原型视觉和交互一致，并产出可一键初始化的完整 SQL 文件。

**Architecture:** 后端采用 Spring Boot 2.7 + MyBatis Plus + 动态多数据源 + Redis + MinIO；前端采用 React JSX + Ant Design + 设计令牌体系。功能实现以 `04.功能规格_V1.0.md` 为唯一功能口径，以 `06.前端实现计划.md` 为样式与主题口径，以 `prototypes` 为页面结构与交互参考。

**Tech Stack:** Spring Boot 2.7, MyBatis Plus, Redis, MinIO, MySQL 5.7, React JSX, Vite, Ant Design。

---

## 0. 基线与约束

**范围冻结**
- V1 必做：`04.功能规格_V1.0.md` 中 `V1必做` 条目。
- V1 可选：仅在 V1 必做全部通过后进入。
- V2：不进入本轮开发。

**完成标准**
- 后端 API、前端页面、权限、审计、日志、配置、文件、消息均按功能规格通过联调。
- 原型对齐：布局、组件、状态色、交互路径与 `prototypes` 保持一致（允许工程化微调，不改变信息结构）。
- 产出完整 SQL 初始化包：一键初始化库结构 + 初始数据 + 索引 + 约束 + 回滚脚本。

## 4. 关键执行规则

- 以 `04.功能规格_V1.0.md` 为功能裁决文件，出现歧义以该文档为准。
- 前端视觉与交互以 `prototypes` 为准，样式变量以 `06.前端实现计划.md` 令牌体系为准。
- 所有数据库对象必须包含租户隔离字段与审计字段（按模块规则）。
- 每个任务结束必须同步 SQL 变更，禁止后补。
- 每个任务至少一次联调验证（前端页面 + 后端接口 + 数据落库）。
